## Overview

Chapter 1 establishes the **baseline security controls** every DevSecOps pipeline should implement before advanced threat modeling or runtime protection.

### Topics covered

| Topic | Focus |
|-------|--------|
| Shift-left security | Early feedback, developer ownership |
| SAST / DAST | Static and dynamic analysis in CI |
| Secrets management | Prevent credential leakage in builds |

### Learning outcomes

After completing this chapter, readers can design a pipeline with automated security stages, clear failure policies, and audit-friendly evidence for compliance reviews.
