RHCSA exam candidates should be able to accomplish the tasks below without assistance. These have been grouped into several categories.

## Chapter 1: Understand and use essential tools

- <strong>Lesson 1.1:</strong> Access a shell prompt and issue commands with correct syntax
- <strong>Lesson 1.2:</strong> Use input-output redirection (>, >>, |, 2>, etc.)
- <strong>Lesson 1.3:</strong> Use grep and regular expressions to analyze text
- <strong>Lesson 1.4:</strong> Access remote systems using SSH
- <strong>Lesson 1.5:</strong> Log in and switch users in multiuser targets
- <strong>Lesson 1.6:</strong> Archive, compress, unpack, and uncompress files using tar, gzip, and bzip2
- <strong>Lesson 1.7:</strong> Create and edit text files
- <strong>Lesson 1.8:</strong> Create, delete, copy, and move files and directories
- <strong>Lesson 1.9:</strong> Create hard and soft links
- <strong>Lesson 1.10:</strong> List, set, and change standard ugo/rwx permissions
- <strong>Lesson 1.11:</strong> Locate, read, and use system documentation including man, info, and files in /usr/share/doc

## Chapter 2: Create simple shell scripts
- <strong>Lesson 2.1:</strong> Conditionally execute code (use of: if, test, [], etc.)
- <strong>Lesson 2.2:</strong> Use Looping constructs (for, etc.) to process file, command line input
- <strong>Lesson 2.3:</strong> Process script inputs ($1, $2, etc.)
- <strong>Lesson 2.4:</strong> Processing output of shell commands within a script

## Chapter 3: Operate running systems
- <strong>Lesson 3.1:</strong> Boot, reboot, and shut down a system normally
- <strong>Lesson 3.2:</strong> Boot systems into different targets manually
- <strong>Lesson 3.3:</strong> Interrupt the boot process in order to gain access to a system
- <strong>Lesson 3.4:</strong> Identify CPU/memory intensive processes and kill processes
- <strong>Lesson 3.5:</strong> Adjust process scheduling
- <strong>Lesson 3.6:</strong> Manage tuning profiles
- <strong>Lesson 3.7:</strong> Locate and interpret system log files and journals
- <strong>Lesson 3.8:</strong> Preserve system journals
- <strong>Lesson 3.9:</strong> Start, stop, and check the status of network services
- <strong>Lesson 3.10:</strong> Securely transfer files between systems

## Chapter 4: Configure local storage
- <strong>Lesson 4.1:</strong> List, create, delete partitions on MBR and GPT disks
- <strong>Lesson 4.2:</strong> Create and remove physical volumes
- <strong>Lesson 4.3:</strong> Assign physical volumes to volume groups
- <strong>Lesson 4.4:</strong> Create and delete logical volumes
- <strong>Lesson 4.5:</strong> Configure systems to mount file systems at boot by universally unique ID (UUID) or label
- <strong>Lesson 4.6:</strong> Add new partitions and logical volumes, and swap to a system non-destructively

## Chapter 5: Create and configure file systems
- <strong>Lesson 5.1:</strong> Create, mount, unmount, and use vfat, ext4, and xfs file systems
- <strong>Lesson 5.2:</strong> Mount and unmount network file systems using NFS
- <strong>Lesson 5.3:</strong> Configure autofs
- <strong>Lesson 5.4:</strong> Extend existing logical volumes
- <strong>Lesson 5.5:</strong> Create and configure set-GID directories for collaboration
- <strong>Lesson 5.6:</strong> Diagnose and correct file permission problems

## Chapter 6: Deploy, configure, and maintain systems
- <strong>Lesson 6.1:</strong> Schedule tasks using at and cron
- <strong>Lesson 6.2:</strong> Start and stop services and configure services to start automatically at boot
- <strong>Lesson 6.3:</strong> Configure systems to boot into a specific target automatically
- <strong>Lesson 6.4:</strong> Configure time service clients
- <strong>Lesson 6.5:</strong> Install and update software packages from Red Hat Network, a remote repository, or from the local file system
- <strong>Lesson 6.6:</strong> Modify the system bootloader

## Chapter 7: Manage basic networking
- <strong>Lesson 7.1:</strong> Configure IPv4 and IPv6 addresses
- <strong>Lesson 7.2:</strong> Configure hostname resolution
- <strong>Lesson 7.3:</strong> Configure network services to start automatically at boot
- <strong>Lesson 7.4:</strong> Restrict network access using firewall-cmd/firewall

## Chapter 8: Manage users and groups 
- <strong>Lesson 8.1:</strong> Create, delete, and modify local user accounts
- <strong>Lesson 8.2:</strong> Change passwords and adjust password aging for local user accounts
- <strong>Lesson 8.3:</strong> Create, delete, and modify local groups and group memberships
- <strong>Lesson 8.4:</strong> Configure superuser access

## Chapter 9: Manage security
- <strong>Lesson 9.1:</strong> Configure firewall settings using firewall-cmd/firewalld
- <strong>Lesson 9.2:</strong> Manage default file permissions
- <strong>Lesson 9.3:</strong> Configure key-based authentication for SSH
- <strong>Lesson 9.4:</strong> Set enforcing and permissive modes for SELinux
- <strong>Lesson 9.5:</strong> List and identify SELinux file and process context
- <strong>Lesson 9.6:</strong> Restore default file contexts
- <strong>Lesson 9.7:</strong> Manage SELinux port labels
- <strong>Lesson 9.8:</strong> Use boolean settings to modify system SELinux settings
- <strong>Lesson 9.9:</strong> Diagnose and address routine SELinux policy violations

## Chapter 10: Manage containers
- <strong>Lesson 10.1:</strong> Find and retrieve container images from a remote registry
- <strong>Lesson 10.2:</strong> Inspect container images
- <strong>Lesson 10.3:</strong> Perform container management using commands such as podman and skopeo
- <strong>Lesson 10.4:</strong> Build a container from a Containerfile
- <strong>Lesson 10.5:</strong> Perform basic container management such as running, starting, stopping, and listing running containers
- <strong>Lesson 10.6:</strong> Run a service inside a container
- <strong>Lesson 10.7:</strong> Configure a container to start automatically as a systemd service
- <strong>Lesson 10.8:</strong> Attach persistent storage to a container

Reference: [Red Hat EX200](https://www.redhat.com/en/services/training/ex200-cyan-hat-certified-system-administrator-rhcsa-exam)
