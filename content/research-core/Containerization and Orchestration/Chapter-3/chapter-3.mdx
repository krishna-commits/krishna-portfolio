## Change the root password from boot 
- Reboot the system 
- Press `shift` key continuosly
- Select `Rescue mode` using the `e` key.
- After `quiet` add `rd.break`, and press ctrl+x.
- A command line interface will appear with a blink. Press enter if no texts appear.
- Then write the commmand `mount -o remount,rw /sysroot`
- Then write the command `chroot /sysroot`
- Then write the command to change password `passwd`
- Then write the command `touch /.autorelabel`
- Then exit `exit`
- Then reboot `reboot`

## How to set a password in the bootloader 
The bootloader name of linux is GRUB2 (Grand Unified Bootloader-V2).
Main Configuration File of GRUB2 is `grub.cfg`
```bash /grub.cfg/
[root@assignmentserver grub2]# ls
fonts  grub.cfg  grubenv
```
```bash
[root@assignmentserver grub2]# pwd
/boot/grub2

[root@assignmentserver grub2]# ls
fonts  grub.cfg  grubenv

# To set the password use this command
[root@assignmentserver grub2]# grub2-setpassword
Enter password: 
Confirm password: 

# A new file which stores the password in encrypted format will be stored in the user.cfg file
[root@assignmentserver grub2]# ls
fonts  grub.cfg  grubenv  user.cfg

[root@assignmentserver grub2]# cat user.cfg 
GRUB2_PASSWORD=grub.pbkdf2.sha512.10000.28D545A97CD026F1C99F93F4662F80ADD5C10F9127830DBC7CDF3A9B37599CC7D6FC46A35370FBEDB22FBE0A40AC13A99CEEB8686506919A05E962E6AB6D62A5.82C0C23474754A6A053EAA707AA96CE24B120027AC48D17A0C799F738921FB92B6E12558E377B9AFF27B3DE6DA2D8015DB9B225BECD3697A5E632E981E6208CC

# Now reflect the changes made to user.cfg in the main file grub.cfg
[root@assignmentserver grub2]# grub2-mkconfig >grub.cfg 
Generating grub configuration file ...
Adding boot menu entry for UEFI Firmware Settings ...
done

```

There are two types of Priority
- System Managed Priority `PR` : Cannot be altered. Managed by system internally.
- User Managed Priority `NI` : Can be altered 

## Priority Nice Value Denotation
- Default Priority (Nice Value) : 0 
- Max Priority (Nice Value) : -20
- Min Priority (Nice Value) : +19

## Commands to alter Priority of Processes
- `nice` : It is used to set the Priority on a process at the process initialization.
- `renice` : It is used to alter the Priority on a process at runtime. 

### Example 1: Using renice to alter the priority
```bash
[root@assignmentserver ~]# top 
    PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND                            
    851 root       5   0   458748   8440   6776 S   0.3   0.2   2:06.22 vmtoolsd  

[root@assignmentserver ~]# renice -15 851
851 (process ID) old priority 0, new priority -15

[root@assignmentserver ~]# top 
    PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND                            
    851 root       5 -15  458748   8440   6776 S   0.3   0.2   2:06.22 vmtoolsd                           
```

### Example 2: Using nice to set the priority at initialization
```bash 
[root@assignmentserver ~]# nice -n -5 find /

[root@assignmentserver ~]# top 
    PID USER      PR  NI    VIRT    RES    SHR S  %CPU  %MEM     TIME+ COMMAND                            
   8847 root      15  -5  221872   2432   2176 R   5.0   0.1   0:00.15 find        
```
